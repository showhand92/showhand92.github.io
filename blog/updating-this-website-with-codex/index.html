Updating This Website with Codex

May 16 2026 · 1 min read
#website

I recently started using Codex to update this personal website again. The site is still intentionally simple: a static GitHub Pages site with hand-written HTML, shared CSS, and a few lightweight content sections.

The current structure is organized around a small set of stable pages. The homepage remains an about page with education, publications, news, and selected links. The blog section is for short notes like this one. The travel section is for itinerary planning pages, including maps, photo references, and route notes.

For now I am keeping the site static instead of adding a full backend. This keeps deployment simple and makes it easy to edit content locally, commit changes, and publish through GitHub Pages. If I need online editing, login, or database-backed content later, I may revisit the architecture.

Going forward, I plan to use the blog for writing and technical notes, and the travel section for trip plans. The first travel collection is a Kyushu itinerary covering Kumamoto, Kagoshima, Sakurajima, and Sengan-en.
