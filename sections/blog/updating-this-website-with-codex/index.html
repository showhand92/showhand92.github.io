Updating This Website with Codex

How this static GitHub Pages site is organized, and how Blog and Travel will be updated going forward.

May 16 2026 1 min read

I recently started using Codex to update this personal website again. The site is still intentionally simple: a static GitHub Pages site with hand-written HTML, shared CSS, and a few lightweight content areas.

The current structure is organized around a small set of stable pages. The homepage remains an about page with education, publications, news, and selected links. Blog is for short notes like this one. Travel is for itinerary planning pages, including maps, photo references, and route notes.

For now I am keeping the site static instead of adding a full backend. This keeps deployment simple and makes it easy to edit content locally, commit changes, and publish through GitHub Pages. If I need online editing, login, or database-backed content later, I may revisit the architecture.

Going forward, I plan to use Blog for writing and technical notes, and Travel for trip plans. The first travel collection is a Kyushu itinerary covering Kumamoto, Kagoshima, Sakurajima, and Sengan-en.
